# 共通スニペット集

複数の成果物テンプレートで参照される、書式起因ループを最も発生させやすい記法の**正準例示**。
各成果物テンプレはこのディレクトリの該当スニペットへのリンクを含み、担当エージェントは
テンプレとスニペット両方を Read してから書き始める。

## スニペット一覧

| ファイル | 用途 | 対応する書式起因high の主要パターン |
|---|---|---|
| `decision_token_example.md` | `[D:id=value]` 決定参照トークンの正しい例示（コードフェンス必須） | 説明文中の例示がフェンス外にあり ontology_scan が実参照と誤認するパターン |
| `req_marker_example.md` | 要件定義書の `**REQ-NNN**` 太字マーカー付き見出し | 太字マーカー欠落で trace_scan の req_set が空になり試験計画のトレースが全件 dangling 化するパターン |
| `trace_line_example.md` | 試験計画の `**トレース**:` 行と上流ID規約 | トレース行の欠落・旧記法・上流ID型ミスマッチ |
| `evidence_ref_example.md` | `[E_n]` エビデンス参照の書式 | 実装ソースに設計書エビデンスへの根拠コメントが欠落するパターン |

## 追加基準

新規スニペットを追加するのは、以下の全条件を満たす場合のみ:
1. 2つ以上のテンプレから参照される共通記法である
2. 書式起因の high が繰り返し観測された記法である
3. コードフェンス／文字マーカー／記法の3要素のいずれかが原因

上記に該当しない場合は、テンプレ内に直接例示を埋める（過剰共通化を避ける）。
